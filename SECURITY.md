# Security Policy

The security of Javascomp is a priority. We appreciate the efforts of security researchers and developers who work to improve the integrity of this project.

## Supported Versions

As the project is in active development (alpha/pre-alpha), only the latest version on the `main` branch is considered supported for security fixes.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **DO NOT** open a public issue.

Please send a detailed email to the maintainers (insert contact email here or use GitHub's private reporting feature if enabled).

We will try to respond as quickly as possible to address the issue.

## Disclosure Process

1.  The report is received and analyzed.
2.  The vulnerability is confirmed.
3.  A patch is developed and tested.
4.  The patch is released.
5.  The vulnerability is publicly disclosed after a safe period for updates.
